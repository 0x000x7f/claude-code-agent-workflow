# output-style — 応答の出力様式（全機体共通ルール）

このファイルは `~/.claude/rules/` に配布され、Desktop / Laptop など全機体でハーネスが毎セッション自動読込する。
機体固有の値（ユーザー名・絶対パスの接頭辞）を直書きしないこと（Desktop=`C:\Users\User`, Laptop=`C:\Users\sawa3`）。

## ファイル参照リンク（チャット応答）

ユーザーへの応答でファイルに言及するときは、**クリックして開ける Markdown リンク**にする。開く先は Claude Desktop 内である必要はなく、OS 既定アプリで開けばよい。

1. **href はスラッシュ区切りの絶対パス**。Windows のバックスラッシュ `\` は Markdown のエスケープ文字なので href に入れると壊れてクリック不能になる。
   - ❌ `[file.md](C:\Users\...\file.md)` — 壊れる
   - ✅ `[C:/Users/.../file.md](C:/Users/.../file.md)`
2. **表示テキスト（ラベル）も絶対パスにする**。ファイル名だけをラベルにすると href の絶対パスが見えない。必要なら `:行番号` を付ける。
3. PR / Issue はフル URL の Markdown リンクにする（`#123` のような裸参照にしない）。

## アーティファクト・生成ドキュメント内のパス

- アーティファクト（HTML ウィジェット等）や生成した Markdown では、リンクが**クリック開き**に対応するとは限らない（`file://` はレンダラ／ブラウザに弾かれることがある）。そのため **フル絶対パスを可視テキストとして必ず出す**（最低限コピペで辿れる）。クリック可能にできる面ではリンクにもする。
- 「パスをファイル名だけに省略して隠す」ことをしない。アーティファクトでもパスは表示する。

## 同期ファイル内のパス（vault / dotfiles など複数機体で共有される .md）

- 機体固有の絶対パス（`C:\Users\User\…`）を**直書きしない** — Laptop（`C:\Users\sawa3\…`）で壊れる。
- 代わりに **`$env:USERPROFILE\…`（ポータブルな絶対パス）** か **リポジトリ相対パス**を使う。
- 上の「チャットは絶対パス可視化」ルールは**私の応答**に適用するもので、同期ドキュメントの中身には持ち込まない（面ごとに使い分ける）。

## 出自メモ

- 2026-07-08、Desktop の記憶層にしかなかった「絶対パス可視リンク」規約を、両機共有のためこのルールへ昇格。
- 「公式Docs が絶対パス必須」ではない（code.claude.com は @参照が relative/absolute どちらも可・参照は Ctrl+Click で開ける、と述べるのみ）。これはユーザー運用要望として採用したルール。
