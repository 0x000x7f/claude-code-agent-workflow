# output-style — 応答の出力様式（全機体共通ルール）

このファイルは `~/.claude/rules/` に配布され、Desktop / Laptop など全機体でハーネスが毎セッション自動読込する。
機体固有の値（ユーザー名・絶対パスの接頭辞）を直書きしないこと（Desktop=`C:\Users\User`, Laptop=`C:\Users\sawa3`）。

## ファイル参照（チャット応答）— 「開ける」を最優先に

前提（実測・2026-07-08）: Claude Desktop / アーティファクト面では Markdown リンクの**クリック開きが保証されない**。ハーネスがクリック可能化するのは「作業ディレクトリからの相対パス」の href だけの可能性が高く、`C:/…` 絶対パスや `file://` は開けないことがある。よって「絶対パス可視化」と「クリック一発で開く」は両立しにくい。

**方針: リンクの成否に依存せず、ユーザーが自分で開けるようにする。開くアプリはユーザーが選ぶ（Explorer 経由）。**

ファイルに言及するときは、次の2つを両方出す:

1. **絶対パスの可視表示**（コピペで辿れる。ファイル名だけに省略しない）。
2. **Explorer で開くコピペコマンド**（コードブロックで。バックスラッシュ可・Markdown エスケープの影響を受けない）。ファイルはフォルダを開いて選択状態にし、ユーザーが右クリック→プログラムから開く で自分で解決できる:
   ```powershell
   explorer /select,"C:\絶対\パス\file.md"     # ファイルを Explorer で選択表示
   ii "C:\絶対\パス\folder"                       # フォルダ/ファイルを既定アプリで開く
   ```
   - パスに空白があっても `"` で囲めば動く。ファイルを選択表示したいなら `/select,`。
3. 補助として Markdown リンクを付けてもよいが、それを唯一の導線にしない（クリックで開けない面があるため）。
4. PR / Issue はフル URL の Markdown リンク（Web は確実に開くため。裸の `#123` は不可）。

## アーティファクト・生成ドキュメント内

- クリック開きは面依存で保証されない。**フル絶対パスを可視テキストで必ず出す**＋上記の `explorer /select,"…"` コマンドを併記して、ユーザーが Explorer 経由で開けるようにする。パスをファイル名だけに省略しない。

## 同期ファイル内のパス（vault / dotfiles など複数機体で共有される .md）

- 機体固有の絶対パス（`C:\Users\User\…`）を**直書きしない** — Laptop（`C:\Users\sawa3\…`）で壊れる。
- 代わりに **`$env:USERPROFILE\…`（ポータブルな絶対パス）** か **リポジトリ相対パス**を使う。
- 上の「チャットは絶対パス可視化」ルールは**私の応答**に適用するもので、同期ドキュメントの中身には持ち込まない（面ごとに使い分ける）。

## 出自メモ

- 2026-07-08、Desktop の記憶層にしかなかった「絶対パス可視リンク」規約を、両機共有のためこのルールへ昇格。
- 「公式Docs が絶対パス必須」ではない（code.claude.com は @参照が relative/absolute どちらも可・参照は Ctrl+Click で開ける、と述べるのみ）。これはユーザー運用要望として採用したルール。
