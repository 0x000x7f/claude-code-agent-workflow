## 目的

コードベース全体を調査し、Phase 分けされたリファクタリング計画を作る。

## 進め方

1. 大量のファイル読解・検索は Explore サブエージェントに委任し、メイン会話には要約だけ戻す（メイン文脈を汚さない）
2. ディレクトリ構造と主要コンポーネントの責務を整理する
3. 依存関係・データフロー・設定ファイルの関係を把握する
4. 重複、責務混在、命名不統一、テスト不足、セキュリティ上危険な箇所を分類する
5. 変更リスクが低い順に Phase 1〜5 の改善計画を作る

## 禁止事項

- ソースコードの編集（成果物 `docs/refactor-plan.md` の作成のみ許可）
- 大規模な一括変更の提案
- 推測と確認済み事実を混ぜること（未確認事項は明示的に分ける）

## 成果物: docs/refactor-plan.md

以下の構成で `docs/refactor-plan.md` に書き出す。会話内だけに残さないこと（Codex/Cursor 等の他ツール・別セッションからも参照されるため、この文書だけで理解できる自己完結性を持たせる）:

1. 全体概要
2. アーキテクチャ推定（確認済み事実と推測を区別）
3. 問題一覧（分類つき）
4. Phase 別リファクタ計画 — 各 Phase に: 目的 / 対象ファイル / 想定差分 / テスト方法 / ロールバック方法
5. 最初に実装すべき小さい PR 案
6. 未確認事項
