# リファクタリング計画（例）

> `/analyze-codebase` が生成する `docs/refactor-plan.md` のサンプル。
> 架空の小さな CLI ツール（Rust）を題材にしている。

## 1. 全体概要

ログ集計 CLI。`src/main.rs` に引数解析・集計ロジック・出力整形が同居しており、
テストは集計ロジックの一部のみ。ルートに実験用の一時ファイルが散在している。

## 2. アーキテクチャ推定

- 確認済み: `main.rs`（約600行）にすべての責務が集中。外部依存は `clap` と `serde` のみ
- 推測: 出力整形は CSV/JSON の2系統に分岐しており、将来フォーマット追加の意図がある（未確認）

## 3. 問題一覧

| 分類 | 内容 |
|---|---|
| 責務混在 | 引数解析・集計・出力が単一ファイル |
| テスト不足 | 出力整形に対するテストがゼロ |
| 整理不足 | ルートに `temp_*.txt` が10個以上 |

## 4. Phase 別リファクタ計画

### Phase 1: 一時ファイルの整理（リスク最小）

- 目的: ルートの `temp_*.txt` を削除し `.gitignore` に追加
- 対象: リポジトリルート、`.gitignore`
- 想定差分: ファイル削除のみ、コード変更なし
- テスト: `cargo test`（既存テストが通ることのみ確認）
- ロールバック: `git revert` で即時復元可能

### Phase 2: 出力整形の分離

- 目的: `main.rs` から出力整形を `src/format.rs` に切り出す（挙動変更なし）
- 対象: `src/main.rs`, `src/format.rs`（新規）
- 想定差分: 関数移動＋`mod` 宣言。公開 API 変更なし
- テスト: `cargo test` ＋ 移動した関数への単体テスト追加
- ロールバック: 単一コミットを revert

### Phase 3: 集計ロジックの分離とテスト拡充

- 目的: 集計を `src/aggregate.rs` に分離し、境界値テストを追加
- 対象: `src/main.rs`, `src/aggregate.rs`（新規）
- 想定差分: 関数移動＋テスト追加
- テスト: `cargo test`（新規テスト含む）
- ロールバック: 単一コミットを revert

## 5. 最初に実装すべき小さい PR 案

Phase 1（一時ファイル整理）。コード変更ゼロでレビュー負荷が最小のため。

## 6. 未確認事項

- `temp_*.txt` のうち現役で参照されているものがないか（Phase 1 実装前に grep で確認）
- 出力フォーマット追加の予定有無（Phase 2 の切り出し粒度に影響）
