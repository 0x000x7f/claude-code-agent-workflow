# HANDOFF: format-module-extraction（例）

> `/handoff format-module-extraction` が生成する自己完結ハンドオフ文書のサンプル。
> このセッションを知らない実装者（Codex / Cursor / 別セッション / 人間）が、
> この文書だけで作業を完了できることが品質基準。

## 1. 目的

`src/main.rs` から出力整形ロジックを `src/format.rs` に切り出す。挙動変更は一切しない。

## 2. 背景・調査結果

- 確認済み: 出力整形は `format_csv()`（main.rs 320-388行付近）と `format_json()`（390-441行付近）の2関数。どちらも `&[Record]` を受け取り `String` を返す純粋関数
- 確認済み: 両関数を呼ぶのは `run()` 内の1箇所ずつのみ
- 推測ではない事実のみ記載。行番号は多少前後する可能性があるため関数名で特定すること

## 3. 設計・方針

- `src/format.rs` を新規作成し、2関数と関連する `use` を移動する
- `main.rs` に `mod format;` を追加し、呼び出し側は `format::format_csv(...)` に変更する
- 関数シグネチャ・出力文字列は1バイトも変えない

## 4. 対象ファイル

- 変更可: `src/main.rs`, `src/format.rs`（新規）
- 読むだけ: `tests/`, `Cargo.toml`
- 触ってはいけない: 上記以外のすべて

## 5. 制約・禁止事項

- 挙動変更禁止（リファクタのみ）
- 関数のリネーム・シグネチャ変更・「ついで」の整理は禁止
- 依存クレートの追加・更新は禁止
- フォーマッタの一括適用は禁止（差分を移動分だけに保つ）

## 6. テストコマンド

```bash
cargo test          # 全テストが既存と同じ結果で通ること
cargo build         # 警告が増えていないこと
```

## 7. 完了条件

- [ ] `src/format.rs` が存在し、2関数が移動されている
- [ ] `main.rs` から該当関数の定義が消え、呼び出しが `format::` 経由になっている
- [ ] `cargo test` が全件パス
- [ ] 差分が対象ファイル2つに収まっている

## 8. レビュー方法

完了後、元のリポジトリで `/review-diff` により検収される。
検収観点: 挙動不変・差分の最小性・禁止事項違反の有無。
request changes の場合は指摘事項と差分を添えて差し戻される。
